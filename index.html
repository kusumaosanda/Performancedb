KPI Definition

Fulfillment SLA Reference — Wholesale Service Agreement (WSA) Telkom × Telkomsel

Source
WSA 2023
+ 3 Addenda
The Fulfillment KPIs shown across this dashboard (TTI, FFG, TTR FFG, PSRE, FFGGAUL) are contractually defined in Lampiran III — Jaminan Tingkat Layanan, §1 "Fulfillment" of the Wholesale Agreement between PT Telkom Indonesia and PT Telekomunikasi Selular, and refined by three subsequent addenda (Berita Acara Kesepakatan, or "BAK"). This page focuses only on Fulfillment content — the dashboard does not currently surface Assurance, Quality, or Billing KPIs, so those categories are intentionally excluded here even where the source documents cover them. Definitions below reflect the latest document (BAK Jan 2025).
🔒 SLA KPI — penalty-bearing (Kredit Layanan) 📊 Monitoring KPI — informational, non-penalized
Definition per KPI — current definitions, per BAK Jan 2025 (latest)
KPI
SLA WSA
TTI Comply SLA · 60%
"Jumlah pelanggan yang berhasil pasang baru (New Sales/PDA) dalam kurun waktu ≤3×24 jam."
A. MANJA→PC (primary) · B. PI→PC (if MANJA null) · C. RE→PC (if PI null) — comply if ≤72 jam
Limit: 3×24 jam. Scope: New Sales IndiHome (PSB) + PDA. Excludes: Produk Orbit, reseller.
Fulfillment Guarantee (FFG) SLA · 40%
"Indikator yang menunjukkan rasio tidak terjadinya gangguan pada pelanggan Layanan TELKOMSEL dalam jangka waktu target hari setelah pasang baru."
Σ PSB tidak terganggu dalam 60 hari / Σ Total PSB × 100%
Limit: 60 hari · nasional per TREG. Excludes: gamas, isu non-teknis, OTT, perangkat rusak, Produk Orbit, reseller, force majeure (wajib dokumentasi), planned maintenance, kesalahan pelanggan.
TTR Fulfillment Guarantee SLA · not scored
"Persentase penyelesaian gangguan untuk pelanggan pasang baru dalam jangka waktu target jam, mulai dari MANJA."
Σ Tiket selesai dalam 3 jam dari MANJA / Σ Total Tiket × 100%
Limit: 3 jam dari MANJA (valid manja <24h open tiket; else measured from open-ticket time). Excludes: wilsus, Orbit, reseller, force majeure, planned maintenance, kesalahan pelanggan.
📊KPI
Monitoring
PS/RE Ratio Monitoring
"% penyelesaian order pasang baru dari seluruh RE, dikurangi order cancel, dalam <7 hari setelah tenggat RE."
[ Σ Total PS (N-1)+(N-2) ] / [ Σ Total RE (N-2) − Σ Cancel RE <7 hari ] × 100%
Limit: 7 hari setelah tenggat RE. New in BAK Jan 2025, Lampiran III §4 "Monitoring."
Gaul FFG Monitoring
"% Pelanggan PSB TIDAK mengalami Gangguan >1x (Gangguan Ulang) dalam periode fulfillment guarantee."
1 − [ (Σ Tiket − Σ Pelanggan Tergangguan) / Σ Tiket Closed ] × 100%
Limit: 60 hari (same window as FFG). Informally tracked at 83.59% in H2 2024; formalized in BAK Jan 2025.
TTI PT2 Comply ("SLA PT-2") Monitoring
⚠ Currently not monitored on this dashboard
"Jumlah pelanggan yang berhasil pasang baru lewat penambahan alpro PT2 dalam <6 hari."
Σ PS PT2 TTI Sesuai Tolak Ukur / ( Σ Total Order PT2 − Cancel <6 hari ) × 100%
Limit: 6 hari ("agree to complete"). New in BAK Jan 2025, Lampiran III §4 "Monitoring." No live dashboard entity is wired up for this KPI yet.

Service Credit Explanation — Kredit Layanan, Lampiran XI (WSA 2023)
Penalti Fulfillment (Pf) = SC Fulfillment × Jumlah Work Order PSB Bulan Berjalan × OTC per LIS
Scoring weights: Fulfillment Guarantee = 40%, TTI Comply = 60%. TTR Fulfillment Guarantee is not included in this formula — it carries its own SLA target but no penalty weight.

Key Observations
1. Targets rose sharply, then plateaued. FFG went 80% → 97.40% → 98.11% → 98.29% (a step-change in 2024, small increments after). TTI Comply went 67% → 92.72% → 93.31% → 93.31% (flat since S-II 2024). TTR FFG went 70% → 80.81% and has stayed flat at 80.81% for three consecutive periods.
2. Kredit Layanan (penalty) scoring only weights 2 of the 3 SLA KPIs. Fulfillment Guarantee = 40%, TTI Comply = 60%. TTR Fulfillment Guarantee has never been part of the financial-penalty formula in any of the 4 documents — it's an SLA target with its own exclusion logic, but doesn't feed the penalty score.
3. The "target" column was repurposed starting BAK Apr 2024. WSA 2023 used a fixed baseline population percentage (80%/70%/67%); from BAK Apr 2024 onward the same field is overwritten each period with that semester's live target — i.e. it reads as "current period target," not "measurement population," post-2023.
4. Exclusion-rule scope has grown monotonically every period — each addendum adds new carve-outs (Produk Orbit → +reseller; +Force Majeure documentation requirement; +planned-maintenance window; +NTE faulty hardware; +customer-caused disruption) without removing prior ones. A flat headline number (e.g. TTR FFG steady at 80.81%) can still reflect a narrower, cleaner measured population than the prior period.
5. Three new Fulfillment KPIs appear only in BAK Jan 2025, under a new "Monitoring" category explicitly separate from the penalty-bearing SLA KPIs (PS/RE Ratio, TTI PT2 Comply, FFG GAUL).
6. BAK Aug 2024 uniquely covers Fulfillment only — no Assurance/Quality Lampiran — unlike the other three documents, which cover all three categories.
7. H2 targets are always deferred to a mid-year addendum. Every H1-covering document (WSA 2023, BAK Apr 2024, BAK Jan 2025) explicitly states the second-half target will be set "setelah proses review realisasi S-I" — so an H2-2025 addendum is expected but was not among the 4 documents reviewed here.

Conversion Logic Reference
🟦 FFG Population
Group all tickets by order_id. Sort each group by trouble_opentime ascending. Pick the first row where flag_exclude_all = 0. If all rows for an order are excluded, that order is skipped.
🟩 TTR FFG Population
All rows from the source file are kept as-is. The SLA formula (date_close − first_cust_assign ≤ 3h, excluding WILSUS and null assign) is applied separately in update_dashboard.py.
🟧 FFG GAUL Population
Filter to flag_exclude_all = 0. In original file order, the first occurrence of each order_id is labelled FFG (comply), any subsequent ticket for the same order is labelled GAUL (not comply / repeat visit).
